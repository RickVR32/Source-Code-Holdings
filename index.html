02 — The Framework

The Observer
Code model

The Source Code Framework was developed from lived human experience — an original, internally consistent model of human awareness. The language, structure, and all terminology are proprietary intellectual property of Source Code Holdings Ltd.

"Everything in existence must coincide with its polar opposite or cease to exist."
The single structural principle from which the entire framework derives. It determines why the framework is shaped the way it is, why its measurement system works the way it does, and why its developmental model follows a spiral rather than a line.

The practical implication: no code, state, or pattern should be eliminated. The goal is never to remove the negative — it is to balance all components so that none dominates to the point of dysfunction.
Layer 01

Conscious layer

The external, observable layer. What a person shows — their behaviour, decisions, communication, and patterns. Ten discrete codes govern this layer.

  • 10 Conscious Source Codes
  • Observable behaviour and output
  • The active Yin-Yang
Layer 02

Subconscious layer

The internal, hidden layer running simultaneously. What drives the conscious layer without the person's awareness — fear, memory, desire, instinct, attachment. Ten codes.

  • 10 Subconscious Source Codes
  • Emotional patterns and loops
  • The inverse Yin-Yang
The gap

Drift

The measurable distance between what a person presents and what is actually running underneath. A person can appear perfectly regulated while running a destabilising subconscious programme.

  • D = |Conscious − Subconscious|
  • Zero = full alignment
  • The primary measurement output
03 — Observer Codes

Not types.
Percentages.

Four Observer Codes present in every person simultaneously, always summing to 100%. The ratio between them determines current state and predicts behavioural output. Zero-Point — fully balanced awareness — is 25% across all four simultaneously.

NARC

The negative percentage

Extractive, self-protective, survival-oriented energy. Not inherently harmful — at 25% it provides necessary self-preservation. When dominant it becomes zero-sum.

  • Polar pair: YARC
  • At 25%: healthy self-interest
  • When dominant: manipulative, zero-sum
YARC

The positive percentage

Giving, other-directed energy. The polar opposite of NARC. At 25% it produces empathy and genuine connection. When dominant, self-abandoning.

  • Polar pair: NARC
  • At 25%: empathy, connection
  • When dominant: self-abandoning
CIST

The balanced percentage

The integration code. Holds NARC and YARC in conscious regulation — does not eliminate either, contains both. The most stable self-aware state available below Zero-Point.

  • Polar pair: Øll
  • At 25%: stable, grounded
  • When dominant: rigid, over-analytical
Øll

The all-and-nothing percentage

Pure potential in both directions simultaneously. At 25% it provides creative intensity and transformational energy. When dominant, total volatility.

  • Polar pair: CIST
  • At 25%: creative intensity
  • When dominant: implosion or explosion

05 — Hardware · TRL 2–3

Four devices.
In development.

Four proprietary devices designed to passively capture environmental and biometric data for integration with the Awareness Architecture framework. All at concept and early prototype stage (TRL 2–3). Full technical specifications available under NDA.

DP-1

Duality Processor

The core processing engine. Processes parallel input streams to identify cognitive state transitions and pattern drift in real time. Currently deployed as a software implementation (the Omniality Engine).

  • Core engine — all framework logic
  • Powers TCE, NES, and ABFS
  • TRL 2 · Concept validated
TCE

Terminal Consciousness Engine

Primary awareness computation unit. Aggregates data from DP-1, ABFS, and NES to produce a unified real-time awareness score. Designed as a self-coaching smart mirror display.

  • Observer Code ratio display
  • Drift score and recalibration path
  • TRL 3 · Early prototype
NES

Neural Engagement System

Awareness on the body. Detects and classifies neural engagement patterns, monitoring drift throughout the day — surfacing signals before they become behavioural events.

  • Continuous real-time drift monitoring
  • Early warning before behaviour changes
  • TRL 2 · Prototype stage
ABFS

Ambient Field Sensor

Room-level group awareness. Passive environmental sensor array designed to detect and map ambient field conditions — modelling the collective state of everyone in a space as an aggregate.

  • Group Observer Code profile

  • TRL 2 · Concept validated
